# Contributing to PROTOCOL ZERO

Welcome to **PROTOCOL ZERO**, the elite Web3 security masterclass. As an open-source maintainer committed to maintaining the highest standards of quality and relevance, we uphold strict guidelines for contributions. Our goal is to preserve the integrity of this franchise while empowering the community to advance Web3 security.

## Pull Request Standards
All contributions must adhere to **elite-level threat intelligence standards**. Submissions are rigorously reviewed to ensure they align with our mission of bridging Web2 VAPT and Web3 smart contract auditing. Only high-quality, original work that demonstrates deep understanding and practical value will be accepted.

### Accepted Submissions
- **Custom Python/Bash reconnaissance tools**: Advanced scripts for on-chain analysis, wallet tracking, or protocol enumeration.
- **PoC exploit scripts**: Functional Proof-of-Concept exploits using Hardhat or Foundry, targeting real vulnerabilities in DeFi protocols.
- **High-tier disclosed bug bounty writeups**: Detailed technical analyses of complex exploits, including root cause analysis, impact assessment, and remediation strategies.

### Rejected Submissions
- Low-effort AI-generated scripts.
- Broken or non-functional tools.
- Basic beginner CTF walkthroughs or trivial exercises.
- Off-topic content unrelated to Web3 security.

## Mandatory Pull Request Template
All pull requests must include the following information in the description:

### Tool/Script Name
[Provide the name of your submission.]

### Purpose
[Explain the purpose and relevance to Web3 security. Highlight how it advances threat intelligence or exploitation techniques.]

### Usage Instructions
[Provide clear, step-by-step instructions on how to use the tool/script. Include prerequisites, dependencies, and example commands.]

### Proof of Execution
[Include evidence that the tool/script works. This may include screenshots, logs, or a link to a test execution. For exploits, demonstrate successful execution in a controlled environment.]

Pull requests missing any of these elements will be closed without review.

## How to Contribute
1. Fork the repository.
2. Create a new branch for your contribution.
3. Ensure your code is well-documented and follows best practices.
4. Submit a pull request with the required template filled out.
5. Await review from maintainers.

Thank you for upholding the standards of PROTOCOL ZERO. Together, we secure the decentralized frontier.
